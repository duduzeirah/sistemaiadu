Crie um sistema web completo chamado “IADU”, projetado para rodar diretamente no navegador e ser hospedado no GitHub Pages (sem necessidade de backend). OBJETIVO: Criar um sistema extremamente simples, intuitivo e funcional, onde qualquer pessoa consiga usar sem explicação. TECNOLOGIAS: * HTML * CSS * JavaScript puro (sem frameworks) * Armazenamento usando localStorage (para salvar dados do usuário no navegador) ESTRUTURA DO PROJETO: * index.html * style.css * script.js FUNCIONAMENTO GERAL: O sistema deve funcionar como um “gerador de soluções simples”, onde o usuário escolhe um problema e o sistema exibe automaticamente um módulo pronto para uso. IMPORTANTE: * Todos os módulos devem estar implementados * Apenas um módulo deve aparecer por vez * Não usar backend * Tudo deve funcionar offline após carregamento inicial INTERFACE: 1. TELA INICIAL: * Tela limpa e minimalista * Texto central: “O que você quer resolver hoje?” * Três botões grandes: * “Controlar dinheiro” * “Controlar vendas” * “Controlar estoque” 2. TRANSIÇÃO: * Ao clicar em uma opção, mostrar uma barra de progresso simples (simulada com JavaScript) * Após o carregamento, esconder a tela inicial e exibir o módulo escolhido 3. SISTEMA MODULAR: * Cada módulo deve estar em uma “section” * Usar JavaScript para mostrar/ocultar módulos * Nunca exibir mais de um módulo ao mesmo tempo MÓDULOS: MÓDULO FINANCEIRO: * Input: valor de entrada * Input: valor de saída * Botão: adicionar * Lista dinâmica com histórico * Cada item com botão de editar e excluir * Cálculo automático: * total de entradas * total de saídas * saldo atual MÓDULO VENDAS: * Input: nome do produto * Input: cliente * Input: valor * Input: data * Botão: adicionar venda * Lista de vendas * Cada item com editar e excluir * Cálculo automático: * total de vendas * faturamento total MÓDULO ESTOQUE: * Input: nome do produto * Input: quantidade * Input: valor unitário * Botão: adicionar produto * Lista de produtos * Cada item com editar e excluir * Cálculo automático: * quantidade total * valor total em estoque ARMAZENAMENTO: * Usar localStorage para salvar: * dados financeiros * vendas * estoque * Os dados devem persistir mesmo após recarregar a página EXPERIÊNCIA DO USUÁRIO: * Interface extremamente simples * Botões grandes * Texto claro * Sem menus complexos * Sem necessidade de login DESIGN: * Visual moderno e minimalista * Fundo claro * Elementos bem espaçados * Foco em clareza, não em complexidade EXTRAS IMPORTANTES: * Botão para voltar à tela inicial * Animações leves (fade ou transição simples) * Código bem organizado e comentado RESULTADO FINAL: Um sistema funcional onde o usuário entra, escolhe uma opção e começa a usar imediatamente, com dados sendo salvos no navegador. Nome do sistema deve aparecer como: “IADU”
